# US-5.1: Emitter Infrastructure

## Objective

Establish the foundational Swift infrastructure for the v2 code emitter: add
`swift-syntax` as a dependency, define the output model (`EmittedFiles`), and wire
up both the native Swift emitter (`SwiftSyntaxEmitter`) and the template-based
emitter (`MustacheEmitter`) so that every subsequent emitter story has a clean,
uniform base to build on.

## Background

The v1 emitter used string interpolation, which produces syntactically invalid
output when names are ill-formed, indentation drifts, or a helper forgets a
comma. The v2 emitter replaces this with two orthogonal paths:

1. **`SwiftSyntaxEmitter`** — builds a fully typed Swift syntax tree using
   `swiftlang/swift-syntax`, then serialises it with `BasicFormat`. Because the
   output is a `SourceFileSyntax` value, the Swift parser rejects malformed nodes
   before a single character is written to disk.

2. **`MustacheEmitter`** — renders a Mustache template (`swift.mustache` for
   Swift, user-supplied templates for other targets) using
   `hummingbird-project/swift-mustache`. This path is intentionally secondary:
   it exists for customisation and non-Swift targets, not as the Swift
   production path.

Both emitters consume the same v2 `UrkelFile` AST and produce the same
`EmittedFiles` output type. Downstream consumers (CLI, plugin, tests) are
agnostic to which emitter ran.

## Output Model

```
EmittedFiles
  stateMachine : String   — XxxMachine.swift (phase markers, struct, extensions, state enum)
  client       : String   — XxxClient.swift  (runtime builder, client struct)
  dependency   : String   — XxxClient+Dependency.swift (DependencyKey, DependencyValues)
```

A generated file header is prepended to every emitted string:

```swift
// Generated by Urkel — do not edit.
// Source: <machine-name>.urkel
```

## Acceptance Criteria

* **Given** `Package.swift`, **when** the package is resolved, **then**
  `swift-syntax` (≥ 601.0.0) is a resolved dependency of the `Urkel` library
  target.

* **Given** a minimal `UrkelFile` (name `"Toggle"`, no context, two states, one
  transition), **when** `SwiftSyntaxEmitter().emit(file:)` is called, **then**
  it returns a non-empty `EmittedFiles` where every field starts with the file
  header comment.

* **Given** the same `UrkelFile` and a valid `swift.mustache` template string,
  **when** `MustacheEmitter().render(file:templateString:)` is called, **then**
  it returns a non-empty `String`.

* **Given** an invalid Mustache template string, **when** `render` is called,
  **then** it throws `MustacheEmitterError.invalidTemplate`.

* **Given** `SwiftSyntaxEmitter().emit(file:)`, **when** the resulting
  `stateMachine` string is fed back into `swiftParser.parse(source:)` (from
  `swift-syntax`), **then** it parses without errors — every emitted file is
  syntactically valid Swift.

## Implementation Details

### Package.swift

```swift
.package(url: "https://github.com/swiftlang/swift-syntax", from: "601.0.0"),
```

Add `SwiftSyntax`, `SwiftSyntaxBuilder` products to the `Urkel` library target.

### Sources/Urkel/Emitter/EmittedFiles.swift

```swift
/// The three Swift source files produced for a single `.urkel` machine.
public struct EmittedFiles: Sendable {
    /// `XxxMachine.swift` — phase markers, state machine struct, transition
    /// extensions, combined state enum.
    public let stateMachine: String
    /// `XxxClient.swift` — runtime builder and client struct.
    public let client: String
    /// `XxxClient+Dependency.swift` — DependencyKey + DependencyValues.
    public let dependency: String
}
```

### Sources/Urkel/Emitter/EmitterProtocol.swift

```swift
/// Common contract for all Urkel code emitters.
public protocol UrkelEmitter {
    func emit(file: UrkelFile) throws -> EmittedFiles
}
```

## Testing Strategy

* `Tests/UrkelTests/EmitterTests/EmitterInfrastructureTests.swift`
* Construct a minimal `UrkelFile` (Toggle); assert `emit(file:)` returns
  non-nil, non-empty `EmittedFiles`.
* Assert each field starts with `"// Generated by Urkel"`.
* Parse the `stateMachine` field with `SwiftParser.parse(source:)` and assert
  no `Diagnostic` with `severity: .error`.
* Test `MustacheEmitter.render` with a trivial `"{{machineName}}"` template
  and assert the output is the machine name.
* Test that an invalid template throws `invalidTemplate`.
