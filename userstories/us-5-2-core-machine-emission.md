# US-5.2: Core Machine Emission

## Objective

Emit the structural skeleton of a flat state machine: a **phase namespace**,
a **move-only generic machine struct**, **per-phase transition extensions** with
`consuming func` methods, and a **combined state enum** with borrowing accessors
and delegating transition methods. This story covers machines that have a context
type, no guards, no actions beyond the transition hooks, no state-carried data,
and no output events — the baseline that every more advanced story extends.

## Background

### Why `~Copyable`?

A machine that can be copied silently breaks the typestate guarantee: after
calling `m.start()`, the caller still holds `m` and could call `m.start()`
again, entering the same transition twice. `~Copyable` (`consuming` / `borrowing`
semantics) makes double-transition a **compile error**. The move is the proof.

### Why phantom types?

`FolderWatchMachine<Running>` is a different type from
`FolderWatchMachine<Idle>`. The compiler enforces that `.stop()` is only
callable in the `Running` phase. There is no runtime check — the constraint is
erased to a single struct layout at compile time.

### Phase namespace

v1 emitted flat top-level enums (`FolderWatchStateIdle`,
`FolderWatchStateRunning`). v2 nests them inside a single `XxxPhase` enum. This:
- Eliminates global name collisions across machines.
- Makes autocomplete show `FolderWatchPhase.` as a clean namespace.
- Mirrors how Swift stdlib namespaces things (e.g. `AttributedString.Runs`).

## Input DSL

```
machine FolderWatch: FolderContext

@states
  init(directory: URL, debounceMs: Int) Idle
  state Running
  final Stopped

@transitions
  Idle    -> start -> Running
  Running -> stop  -> Stopped
```

## Generated Output

### `FolderWatchMachine.swift`

```swift
// Generated by Urkel — do not edit.
// Source: folderwatch.urkel

import Dependencies
import Foundation

// MARK: - FolderWatch Phases

/// Phase markers for FolderWatchMachine.
/// Use `FolderWatchPhase.Xxx` as the phantom `Phase` type argument.
public enum FolderWatchPhase {
    /// The machine's initial phase.
    public enum Idle {}
    public enum Running {}
    /// A terminal phase — the machine cannot transition further.
    public enum Stopped {}
}

// MARK: - FolderWatch State Machine

/// A type-safe, move-only FolderWatch state machine.
///
/// The current phase is encoded in the `Phase` phantom type parameter.
/// Transition methods are only available when the machine is in the
/// matching phase — invalid transitions are compile-time errors.
public struct FolderWatchMachine<Phase>: ~Copyable, Sendable {
    fileprivate let _context: FolderContext
    fileprivate let _start: @Sendable (FolderContext) async throws -> FolderContext
    fileprivate let _stop:  @Sendable (FolderContext) async throws -> FolderContext

    internal init(
        _context: FolderContext,
        _start: @escaping @Sendable (FolderContext) async throws -> FolderContext,
        _stop:  @escaping @Sendable (FolderContext) async throws -> FolderContext
    ) {
        self._context = _context
        self._start   = _start
        self._stop    = _stop
    }

    /// Read-only access to the current context without consuming the machine.
    public borrowing func withContext<R>(
        _ body: (borrowing FolderContext) throws -> R
    ) rethrows -> R {
        try body(_context)
    }
}

// MARK: - FolderWatchMachine<Idle>

extension FolderWatchMachine where Phase == FolderWatchPhase.Idle {
    /// Transitions from `Idle` to `Running`.
    public consuming func start() async throws -> FolderWatchMachine<FolderWatchPhase.Running> {
        let next = try await _start(_context)
        return FolderWatchMachine<FolderWatchPhase.Running>(
            _context: next, _start: _start, _stop: _stop
        )
    }
}

// MARK: - FolderWatchMachine<Running>

extension FolderWatchMachine where Phase == FolderWatchPhase.Running {
    /// Transitions from `Running` to `Stopped`.
    public consuming func stop() async throws -> FolderWatchMachine<FolderWatchPhase.Stopped> {
        let next = try await _stop(_context)
        return FolderWatchMachine<FolderWatchPhase.Stopped>(
            _context: next, _start: _start, _stop: _stop
        )
    }
}

// MARK: - FolderWatch Combined State

/// Runtime-erasure wrapper over all FolderWatch phases.
/// Use this when the current phase is not statically known (e.g., after
/// storing a machine in a `var` or passing it across an async boundary).
public enum FolderWatchState: ~Copyable {
    case idle(FolderWatchMachine<FolderWatchPhase.Idle>)
    case running(FolderWatchMachine<FolderWatchPhase.Running>)
    case stopped(FolderWatchMachine<FolderWatchPhase.Stopped>)

    /// Wraps an initial-phase machine. Convenience for `FolderWatchState.idle(m)`.
    public init(_ initial: consuming FolderWatchMachine<FolderWatchPhase.Idle>) {
        self = .idle(initial)
    }
}

extension FolderWatchState {

    // MARK: Borrowing accessors

    public borrowing func withIdle<R>(
        _ body: (borrowing FolderWatchMachine<FolderWatchPhase.Idle>) throws -> R
    ) rethrows -> R? {
        guard case .idle(let m) = self else { return nil }
        return try body(m)
    }

    public borrowing func withRunning<R>(
        _ body: (borrowing FolderWatchMachine<FolderWatchPhase.Running>) throws -> R
    ) rethrows -> R? {
        guard case .running(let m) = self else { return nil }
        return try body(m)
    }

    public borrowing func withStopped<R>(
        _ body: (borrowing FolderWatchMachine<FolderWatchPhase.Stopped>) throws -> R
    ) rethrows -> R? {
        guard case .stopped(let m) = self else { return nil }
        return try body(m)
    }

    // MARK: Delegating transitions

    /// Attempts `start` from the current state.
    /// No-ops (returns self) in states where `start` is not defined.
    public consuming func start() async throws -> FolderWatchState {
        switch consume self {
        case .idle(let m):    return .running(try await m.start())
        case .running(let m): return .running(m)
        case .stopped(let m): return .stopped(m)
        }
    }

    /// Attempts `stop` from the current state.
    public consuming func stop() async throws -> FolderWatchState {
        switch consume self {
        case .idle(let m):    return .idle(m)
        case .running(let m): return .stopped(try await m.stop())
        case .stopped(let m): return .stopped(m)
        }
    }
}
```

## Acceptance Criteria

* **Given** a `UrkelFile` with a machine name, a context type, flat states, and
  unguarded transitions, **when** `SwiftSyntaxEmitter().emit(file:)` is called,
  **then** `stateMachine` contains:
  - `public enum XxxPhase` with one nested empty enum per state.
  - `public struct XxxMachine<Phase>: ~Copyable, Sendable`.
  - One `fileprivate let` closure property per unique transition event signature.
  - One `extension XxxMachine where Phase == XxxPhase.Xxx` per source state with
    the appropriate `consuming func` transitions.
  - `public enum XxxState: ~Copyable` with one case per state.
  - A `public init(_ initial: consuming XxxMachine<XxxPhase.InitialState>)` on
    the state enum.
  - One `borrowing func withXxx` per state on the state enum extension.
  - One `consuming func` per unique event on the state enum extension.

* **Given** a machine without a context type (`machine Toggle`), **when** emitted,
  **then** no context closure properties are generated; the `~Copyable` struct
  carries only transition hook closures (or none if there are no injectable hooks).

* **Given** a machine where the initial phase has no `init` state parameters
  (US-1.3 is not used), **when** emitted, **then** the machine struct's `init`
  takes only the closure properties.

* **Given** the emitted `stateMachine` string, **when** parsed with
  `SwiftParser.parse(source:)`, **then** no parser errors.

* **Given** the emitted `FolderWatchMachine.start()` method, **when** examined,
  **then** its return type is `FolderWatchMachine<FolderWatchPhase.Running>` —
  not the combined state enum — preserving typestate narrowing.

* **Given** the combined state enum's `.start()` method, **when** called in the
  `.stopped` case (where `start` is not a valid transition), **then** it returns
  `.stopped` unchanged — the combined state enum is a best-effort wrapper that
  never panics.
