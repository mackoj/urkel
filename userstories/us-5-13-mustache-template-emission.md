# US-5.13: Mustache Template Emission

## Objective

Ship a `swift.mustache` template that produces idiomatic Swift output equivalent
to the `SwiftSyntaxEmitter` for core flat machines (US-5.1 ŌĆō US-5.3). This gives
teams a starting point they can fork and customise without touching the emitter
source. Also formalise the `TemplateContext` model that maps a `UrkelFile` to the
dictionary shape Mustache templates consume.

## Background

`swift-syntax`-based emission is the primary, guaranteed-correct path. Mustache
templates are the **customisation escape hatch**: a team targeting a different
import style, a different naming convention, or a different concurrency primitive
(e.g., Combine instead of `AsyncStream`) can drop their own `.mustache` file and
get completely custom output ŌĆö no fork required.

The `TemplateContext` is the stable contract between the emitter and templates.
It must expose every field a template author needs, with consistent naming.

## TemplateContext Shape (top-level dictionary)


## Generated `swift.mustache`

```mustache
// Generated by Urkel ŌĆö do not edit.
// Source: {{machineName}}.urkel
{{#imports}}
import {{.}}
{{/imports}}

// MARK: - {{machineTypeName}} Phases

public enum {{machineTypeName}}Phase {
{{#states}}
    {{#isInitial}}/// The initial phase.
    {{/isInitial}}{{#isTerminal}}/// A terminal phase ŌĆö no further transitions.
    {{/isTerminal}}public enum {{typeName}} {}
{{/states}}
}

// MARK: - {{machineTypeName}} State Machine

public struct {{machineTypeName}}Machine<Phase>: ~Copyable, Sendable {
{{#hasContext}}
    fileprivate let _context: {{contextType}}
{{/hasContext}}
{{#transitions}}{{^isInternal}}{{^isOutputEvent}}
    fileprivate let _{{event}}: @Sendable ({{#hasContext}}{{contextType}}{{#params}}, {{type}}{{/params}}{{/hasContext}}{{^hasContext}}{{#params}}{{^isLast}}{{type}}, {{/isLast}}{{#isLast}}{{type}}{{/isLast}}{{/params}}{{/hasContext}}) async throws -> {{#hasContext}}{{contextType}}{{/hasContext}}{{^hasContext}}Void{{/hasContext}}
{{/isOutputEvent}}{{/isInternal}}{{/transitions}}
}
{{#states}}

// MARK: - {{machineTypeName}}Machine<{{typeName}}>
{{#groupedTransitions}}{{#sourceStateTypeName}}{{typeName}}{{/sourceStateTypeName}}

extension {{machineTypeName}}Machine where Phase == {{machineTypeName}}Phase.{{typeName}} {
{{#transitions}}{{^isInternal}}
    public consuming func {{event}}({{#params}}{{name}}: {{type}}{{^isLast}}, {{/isLast}}{{/params}}) async throws -> {{machineTypeName}}Machine<{{machineTypeName}}Phase.{{toTypeName}}> {
        let next = try await _{{event}}({{#hasContext}}_context{{#params}}, {{name}}{{/params}}{{/hasContext}}{{^hasContext}}{{#params}}{{name}}{{/params}}{{/hasContext}})
        return {{machineTypeName}}Machine<{{machineTypeName}}Phase.{{toTypeName}}>(_context: next, /* closures */)
    }
{{/isInternal}}{{/transitions}}
}
{{/groupedTransitions}}
{{/states}}
```

> The full template handles guards, output events, and actions via additional
> Mustache sections. The snippet above shows the core structure for illustration.

## Acceptance Criteria

* **Given** a `UrkelFile` for `Toggle` (no context, two phases), **when**
  `MustacheEmitter().render(file:templateString:swift.mustache)` is called,
  **then** the output contains `public enum TogglePhase`,
  `public struct ToggleMachine<Phase>: ~Copyable, Sendable`, and
  `extension ToggleMachine where Phase == TogglePhase.Off`.

* **Given** a `TemplateContext` for FolderWatch, **when** the context's `states`
  array is iterated, **then** each entry has `name`, `typeName`, `variableName`,
  `isInitial`, and `isTerminal` populated correctly.

* **Given** a transition with `isOutputEvent: true`, **when** the template
  renders, **then** a `borrowing var` property is emitted instead of a
  `consuming func`.

* **Given** a `TemplateContext` for a machine with guards, **when** the
  `transitions` array is iterated, **then** each guarded transition has a
  non-empty `guards` array with correct `negated` and `isElse` flags.

* **Given** a custom user template `"{{machineName}}"`, **when** rendered,
  **then** the output is the machine name string ŌĆö proving the context is wired.

## Implementation Details

- `TemplateContext` is produced by `MachineAST+TemplateContext.swift`-equivalent
  for the v2 `UrkelFile` AST.
- The template is embedded as a Bundle resource
  (`Sources/Urkel/Templates/swift.mustache`).
- `MustacheEmitter` has a convenience `renderSwift(file:)` that loads the
  bundled template automatically.

## Testing Strategy

* Snapshot-test the rendered output of `renderSwift(file:)` for Toggle and
  FolderWatch fixtures against known-good snapshots.
* Assert each `TemplateState` field is populated for every state.
* Assert `isOutputEvent: true` on output-event transitions.
* Test custom template `"{{machineVariableName}}"` ŌåÆ lowerCamel name.
