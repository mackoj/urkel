# US-5.3: Client & Dependency Emission

## Objective

Emit the `XxxClient` struct, the `XxxClientRuntime` builder, and the
`DependencyKey` / `DependencyValues` conformances that integrate the machine
with `pointfreeco/swift-dependencies`. Together these three artefacts form the
dependency-injection layer: they let callers construct machines without knowing
their internal closure layout, provide a zero-cost `noop` for tests, and register
the client in the global dependency container.

## Background

The client is the **public API surface** for a machine. Callers never hold a
`XxxMachine<Phase>` directly; they hold an `XxxClient` and ask it to construct a
machine when needed. This separation:

- Keeps the machine's closure layout private (`fileprivate`).
- Lets tests swap the whole client with `noop` in one line.
- Lets live implementations be registered once at app start.

`XxxClientRuntime` is a secondary builder that lets advanced callers construct a
live client by providing individual transition closures, rather than implementing
the full factory closure. This is the primary testing/mocking entry point.

## Input DSL (continuing from US-5.2)

```
machine FolderWatch: FolderContext

@states
  init(directory: URL, debounceMs: Int) Idle
  state Running
  final Stopped

@transitions
  Idle    -> start -> Running
  Running -> stop  -> Stopped
```

## Generated Output

### `FolderWatchClient.swift`

```swift
// Generated by Urkel â€” do not edit.
// Source: folderwatch.urkel

import Dependencies
import Foundation

// MARK: - FolderWatch Runtime Builder

/// Provides individual transition hooks; `fromRuntime` assembles them into a
/// `FolderWatchClient`. Prefer this over implementing the factory closure by hand.
public struct FolderWatchClientRuntime: Sendable {
    public typealias StartTransition = @Sendable (FolderContext) async throws -> FolderContext
    public typealias StopTransition  = @Sendable (FolderContext) async throws -> FolderContext

    public let initialContext: @Sendable (URL, Int) -> FolderContext
    public let start: StartTransition
    public let stop:  StopTransition

    public init(
        initialContext: @escaping @Sendable (URL, Int) -> FolderContext,
        start: @escaping StartTransition,
        stop:  @escaping StopTransition
    ) {
        self.initialContext = initialContext
        self.start = start
        self.stop  = stop
    }
}

// MARK: - FolderWatch Client

/// Dependency client for constructing FolderWatch state machines.
///
/// Inject a live implementation via `DependencyValues.folderWatch`; swap in
/// `FolderWatchClient.noop` or a custom `.fromRuntime(â€¦)` in tests.
@DependencyClient
public struct FolderWatchClient: Sendable {
    /// Factory that produces a machine in its initial phase.
    public var makeObserver: @Sendable (URL, Int) -> FolderWatchMachine<FolderWatchPhase.Idle>
}

extension FolderWatchClient {
    /// No-op client whose transitions are identity operations.
    /// Safe to use in previews and unit tests that do not exercise real effects.
    public static var noop: Self {
        Self { directory, debounceMs in
            FolderWatchMachine<FolderWatchPhase.Idle>(
                _context: FolderContext(),
                _start: { $0 },
                _stop:  { $0 }
            )
        }
    }

    /// Assembles a live client from explicit per-transition closures.
    public static func fromRuntime(_ runtime: FolderWatchClientRuntime) -> Self {
        Self { directory, debounceMs in
            let context = runtime.initialContext(directory, debounceMs)
            return FolderWatchMachine<FolderWatchPhase.Idle>(
                _context: context,
                _start: runtime.start,
                _stop:  runtime.stop
            )
        }
    }
}
```

## Acceptance Criteria

* **Given** a `UrkelFile` with init parameters on the `init` state, **when**
  `emit(file:)` is called, **then** `client` contains:
  - `XxxClientRuntime` with `typealias` per unique transition and an
    `initialContext` closure typed with the init param types.
  - `XxxClient` annotated with `@DependencyClient`, with a `makeObserver`
    closure whose parameter list matches the init state params.
  - `noop` that constructs an identity machine (context untransformed).
  - `fromRuntime` that wires all runtime closures to the machine's injected slots.

* **Given** a machine without init parameters, **when** emitted, **then**
  `makeObserver` takes no parameters (`@Sendable () -> XxxMachine<XxxPhase.Init>`).

* **Given** the emitted `XxxClient+Dependency.swift`, **when** compiled, **then**
  `DependencyValues.xxx` resolves to `XxxClient` through the extension.

* **Given** `XxxClient.noop`, **when** its `makeObserver` is called and each
  transition is invoked, **then** no error is thrown and the machine reaches the
  terminal phase without crashing.

* **Given** the emitted files, **when** parsed with `SwiftParser`, **then** no
  parser errors in either `client` or `dependency` fields.

### Noop machine generation

The `noop` machine is built by recursively following the first transition from
each state until a terminal state is reached. Cycles are detected and broken
with `fatalError("Noop encountered cycle at '\(state)' â€” override with a
real implementation")` to catch accidental use of noop in production.
